# 优化 WYSIWYG 编辑模式下的编辑区内容滑动显示体验

## 任务说明

优化 WYSIWYG 编辑器的滚动和布局体验，移除嵌套滚动设计，实现内容自然展开的平铺布局。

## 范围

### 主要修改
- 修改 `src/components/WysiwygEditor.tsx` 文件
- 移除编辑区的固定高度限制 (`max-h-[600px]`)
- 移除编辑区的内部滚动 (`overflow-auto`)
- 优化编辑区布局，让内容自然展开

### 预期结果
- 编辑区内容能够自然展开，无内部滚动条
- 页面滚动流畅，与内容位置同步
- 光标/选择区域定位准确
- 所有编辑功能正常工作
- 响应式适配良好

## 验收标准

### 功能验收
1. 编辑区内容能够自然展开，无内部滚动条
2. 页面滚动流畅，与内容位置同步
3. 光标/选择区域定位准确
4. 所有编辑功能正常工作
5. 响应式适配良好

### 性能验收
1. 页面加载时间正常
2. 内容渲染流畅
3. 大篇幅文档滚动无明显延迟

### 布局验收
1. 编辑区与其他页面元素布局协调
2. 在不同设备上显示一致
3. 整体界面美观且功能完整

## 测试步骤

### 基本功能测试
1. 打开应用程序
2. 进入 WYSIWYG 编辑模式
3. 验证内容是否能够自然展开
4. 检查是否存在内部滚动条
5. 测试页面滚动是否流畅
6. 验证所有编辑功能是否正常工作

### 响应式测试
1. 使用开发者工具模拟不同屏幕尺寸
2. 验证编辑器在桌面端、平板和移动端的布局
3. 测试在小屏幕设备上的显示效果

### 性能测试
1. 输入大量内容以创建大篇幅文档
2. 测试页面滚动性能
3. 验证编辑过程中是否有卡顿
