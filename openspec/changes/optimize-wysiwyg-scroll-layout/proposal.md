# 优化 WYSIWYG 编辑模式下的编辑区内容滑动显示体验

## 背景

当前 WYSIWYG 编辑器采用固定高度的嵌套滚动容器设计，导致内容显示和滚动体验不佳。编辑器内部有自己的滚动区域，与浏览器窗口的滚动行为冲突，影响用户体验。

## 问题分析

### 当前设计的问题

1. **嵌套滚动问题**：编辑器使用固定 `max-h-[600px]` 和 `overflow-auto` 导致内部滚动
2. **内容截断**：长内容会被截断，需要在编辑器内部滚动查看
3. **滚动同步**：编辑器内部滚动与页面滚动不同步
4. **响应式问题**：在不同屏幕尺寸上显示效果不一致
5. **性能影响**：固定高度容器可能影响内容渲染性能

## 目标

优化 WYSIWYG 编辑模式下的内容滑动显示体验，实现：

1. 移除嵌套滚动设计
2. 实现内容自然展开的平铺布局
3. 优化滚动交互体验
4. 提升响应式适配能力
5. 优化渲染性能

## 方案概述

### 核心修改

1. 移除编辑器的固定高度限制和内部滚动
2. 让内容自然展开，利用浏览器的原生滚动
3. 优化布局结构，确保与页面其他元素协调
4. 提升在不同设备上的显示效果

### 预期改进

- 内容显示更完整，无需在编辑器内部滚动
- 滚动体验更自然，与页面其他部分同步
- 响应式适配更好，在移动设备上体验更佳
- 渲染性能提升，内容展开式布局更符合浏览器渲染优化原则
